Frontend (Vite + React) files & commands

Fix the tsc: not found build error

Your earlier log showed sh: 1: tsc: not found. That happens when package.json build calls TypeScript compiler (tsc) but typescript is not installed as a dependency/devDependency. Two ways to fix:

  • If you use TS: install TypeScript locally:

    bash
    npm install --save-dev typescript
# ensure your project has tsconfig.json

  • If you don't use TS: remove tsc from build hooks or fix scripts to use vite only.

Common package.json scripts for Vite:

json
"scripts": {
  "dev": "vite",
  "build": "vite build",
  "preview": "vite preview"
}

If your build script calls something else (like a tsc step), either add typescript to devDependencies or remove that step. (Fixing this resolves the tsc: not found error.) vitejs

Build & test locally

bash
npm ci
npm run build
# serve locally to check the build:
npx serve dist   # or `vite preview`

vercel.json (Vercel SPA rewrites so deep links dont 404)

json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}

Vite + Vercel docs recommend this to enable deep-linking for SPA routes. Vercel+1

If you deploy frontend as a Static Site on Render

Add a Rewrite on the Render static site: Source /* Destination /index.html. Render docs and community threads confirm you need to rewrite all paths to index.html for client-side routing to work. Render+1

  4. Publish Directory: dist

  5. After creation, go to Settings Rewrites (Render UI) and add:

5) Common issues & fixes (directly actionable)

  • sh: 1: tsc: not found
    Cause: your package.json build script runs tsc but typescript isn't installed. Fix: either

    • install TypeScript: npm install --save-dev typescript (and include tsconfig.json), or

    • change the build script to "vite build" and remove extra tsc step.

  • SPA 404 on deep link (e.g., /dashboard)
    Fix: add Render rewrite: /* /index.html. (Render static site Settings Rewrites). That prevents direct-route 404s.

  • 500 after deploy (backend)

    • Check Render logs (Deploys or Logs) they show stack trace.

    • Ensure env vars (DB url, secrets) are set.

    • If DB connections fail, check firewall/whitelist and credentials.

  • Build fails due to node modules

    • Use npm ci for clean install (preferred in CI).

    • If using private packages, add access tokens in environment settings.
